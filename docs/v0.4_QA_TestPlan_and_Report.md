### 1.2 流量主、UV 与「先上线再开广告」

> 与 [微信公众平台](https://mp.weixin.qq.com) → **流量主** 规则一致；具体数值以**后台当前页面**为准。

- **开通流量主**后，方可在后台**创建**激励视频 / 插屏 / Banner 等**真实广告位 ID**；未开通时，「开通」按钮可能为置灰，**属正常情况**。
- 常见**条件 1**：小游戏 / 小程序 **累计独立访客（UV）** 达到平台要求（当前团队侧为 **500 UV** 等，以微信展示为准），且无严重违规记录。未达门槛前**无法**申请真实广告位，**不阻碍**先发布**体验版 / 正式版**跑主流程与云存等业务验收。
- **测试策略分工**：  
  - **UV 未达标、仅用占位 `AD_UNIT_IDS` 的阶段**：重点验收 **玩法、云存、分享、合规**；广告相关用例以 **「加载失败 / 不出贴片」时的降级与流程不卡死** 为主；激励**完整观看后发奖励**、插屏/Banner **真实展示**，可待流量主开通并填入真实 ID 后补测或标 **BLOCKED（待流量主）**。  
  - **UV 达标并开通流量主后**：在后台创建三类广告位，替换 `GameConfig.ts` 中 ID，再完成 §7～§8 的 **PASS** 级验收。
- **上线说明建议**：对外版本公告中可注明「广告变现将在开通流量主并替换广告位后全面启用」，避免用户将「暂无广告」误判为 Bug。

### 1.3 广告位配置（必核）

> `assets/Scripts/Data/GameConfig.ts` 中 **`AD_UNIT_IDS`** 若为占位串（`adunit-xxx…`），在未替换为后台真实广告位 ID 前，微信侧可能**无法出广告**，用例可能表现为加载失败或一直失败 —— 需在报告中注明 **「广告位未配置 / 流量主未开通」** 或 **BLOCKED**，勿误判为逻辑 Bug。

## 3. 测试顺序总览（建议顺序）

1. **Smoke**：启动 → 主菜单 → 进关 → 倒水 → 通关（证明可玩）  
2. **Banner**：主菜单 / 选关 / 结算展示；游戏中 / 暂停 / 设置隐藏  
3. **激励视频**：按场景 **hint / undo / double_coins / free_coins** 分别测  
4. **激励上限与冷却**：与配置一致（见 §5 期望摘要）  
5. **插屏**：第 6 关起按间隔触发；冷却与每日上限  
6. **降级**：关闭广告模块或加载失败时不卡死流程  
7. **存档**：`adStats` 跨天重置；激励/插屏计数合理  
8. **v0.3 回归**：登录与云存、分享（最短路径）  
9. **合规**：设置内无「去广告 / 内购」入口（v0.4 纯广告模式）

## 7. 激励视频 — 四点位

> **完整观看**判定：以 `wx` 回调 `isEnded === true` 为准；否则提示文案见各场景（与 `AdManager.getRewardedFailMessage` 一致）。  
> **次数**：所有激励场景共用 **`adStats.rewardedToday`** 与 **`lastRewardedTime`** 冷却。

### R01 点位 hint（提示）
- **前置**：金币 **少于 50**，且无法用金币买提示  
- **步骤**：点击 **提示** → 在弹窗点 **看视频** → 完整观看 / 中途关闭 各测一次  
- **期望**：  
  - 完整观看：执行提示（高亮建议瓶子），存档写入，`CloudSaveManager.uploadSave` 可异步触发  
  - 未看完：Toast 含 **「视频未完整观看，暂未获得提示。」**（以实际文案为准）  
- **结果**：________  
- **证据**：________  

### R02 点位 undo（撤销）
- **前置**：已用完 **3** 次免费撤销；**videoUndoCredits** 为 0；金币 **少于 10**  
- **步骤**：点击 **撤销** → **看视频** → 完整观看  
- **期望**：获得 **5** 次视频撤销额度并消耗 **1** 次完成本次撤销（与 `VIDEO_REWARD_COUNT` 一致）

## 8. 插屏广告（关卡策略）

### I01 关卡条件
- **说明**：在 **`下一关`**、**返回选关`** 路径上调用 `AdManager.showInterstitial(levelId)`，参数为 **刚通关的 `levelId`**。  
- **期望**：仅当 `levelId` 为 **6、9、12、15…**（从第 **6** 关起每 **3** 关一次）且未超每日上限、满足冷却时尝试展示。  
- **验证建议**：用 GM 或改存档将进度调到 **第 6 关通关**，点 **下一关** / **返回选关**，观察是否尝试插屏。  

### I02 每日上限与冷却
- **期望**：插屏展示成功后 **`interstitialToday`** 递增；达 **20** 次后不再展示；两次展示间隔 **≥ 300** 秒（配置项）
