## 2. 内存泄漏修复 🔴 P0

### 2.1 问题描述

`GameManager.ts` 中所有面板切换、关卡切换、UI 刷新均使用 `removeAllChildren()` 清理旧节点，但**未调用 `destroy()`**。Cocos Creator 中 `removeAllChildren` 仅从父节点移除，节点及其组件（Graphics、Label、事件监听器）仍驻留内存。

**影响范围**：
- `clearView()` — 清空 ViewRoot/PopupLayer/BottleLayer
- `buildBaseLayers()` — 重建基础层
- 所有 `showXxxPanel()` — 面板弹出前清理
- `renderBottles()` — 关卡内重新渲染瓶子
- `showDailyCheckInPanel()` — 签到格子呼吸动画 `repeatForever` tween 未停止

**影响程度**：玩家频繁切换面板/关卡后，内存持续增长，微信小游戏环境可能触发 OOM 或被系统回收。

### 2.2 修复方案

```typescript
// 新增通用安全清理方法（建议放在 UIManager 或 GameManager 底部工具区）
private safeClear(node: Node): void {
    if (!node) return;
    const children = node.children.slice(); // 复制数组避免遍历中修改
    children.forEach(child => {
        // 停止该节点上的所有 tween
        Tween.stopAllByTarget(child);
        child.destroy();
    });
    node.removeAllChildren();
}
```

**替换范围**：

| 方法 | 当前代码 | 替换为 |
|---|---|---|
| `clearView()` | `this.viewRoot?.removeAllChildren()` | `this.safeClear(this.viewRoot)` 等 |
| `buildBaseLayers()` | `child.destroy()` 仅 Background | 所有三层均 `safeClear` |
| `showMainMenu()` 等 10+ 面板 | `this.popupLayer?.removeAllChildren()` | `this.safeClear(this.popupLayer)` |
| `renderBottles()` | `layer.removeAllChildren()` | `this.safeClear(layer)` |

**签到面板特殊处理**：
```typescript
// showDailyCheckInPanel() 中，面板关闭前停止所有呼吸动画
// 当前代码：
tween(cell).repeatForever(tween().to(0.6, { scale: ... }).to(0.6, { scale: Vec3.ONE })).start();
// 需要在 safeClear 中通过 Tween.stopAllByTarget(cell) 停止
```

### 2.3 验收标准

- [ ] 微信开发者工具真机调试，连续切换主菜单→选关→游戏→暂停→设置→返回 20 次，内存不持续增长

## 3. 运行时 Bug 修复 🔴 P0

### 3.1 PourController — 倒水动画高度计算错误

**问题**: `UITransform` 无 `height` 属性，代码中使用 `transform.height` 返回 `undefined`，导致流动条高度为 `NaN`。

**修复**:
```typescript
// PourController.ts 第 11、15、19、20 行
// 旧：transform.height
// 新：transform.contentSize.height

const height = transform.contentSize.height;
// 所有 transform.height 替换为 height
```

**验收标准**:
- [ ] 倒水动画中流动条可见且高度与倾倒格数成正比
- [ ] 无 `NaN` 相关控制台警告

### 3.2 Bottle — 选中/抖动动画冲突

**问题**: `setSelected()` 中 `tween(this.node).stop()` 只能停止同一条 tween 链，`playShake()` 的 tween 是另一个实例，两者同时操作 position 导致错位。

**修复**:
```typescript
// Bottle.ts
// setSelected 和 playShake 前统一停止所有 tween
setSelected(selected: boolean): void {
    Tween.stopAllByTarget(this.node); // 确保停止所有动画
    // ... 原有逻辑
}

playShake(): void {
    Tween.stopAllByTarget(this.node); // 确保停止所有动画
    // ... 原有逻辑
}
```

**验收标准**:
- [ ] 快速连续点击错误瓶子，抖动动画正常结束且位置正确
- [ ] 选中状态下触发抖动，结束后仍保持正确选中偏移

## 4. 云存档合并修复 🟡 P1

### 4.1 Achievements 合并丢失

**问题**: `CloudSaveManager.mergeSaveData()` 遍历 `Object.keys(remote.achievements)`，当 `local` 或 `remote` 中某成就缺失时直接 `return`，导致该成就不被合并。

**修复**:
```typescript
// 遍历 local 与 remote achievements 的并集
const allIds = new Set([
    ...Object.keys(local.achievements),
    ...Object.keys(remote.achievements)
]);
allIds.forEach(id => {
    const localProgress = local.achievements[id];
    const remoteProgress = remote.achievements[id];
    if (!localProgress) {
        base.achievements[id] = remoteProgress;
    } else if (!remoteProgress) {
        base.achievements[id] = localProgress;
    } else {
        // 原有合并逻辑
        base.achievements[id].current = Math.max(localProgress.current, remoteProgress.current);
        base.achievements[id].completed = localProgress.completed || remoteProgress.completed;
        // ...
    }
});
```

### 4.2 合并策略缺陷

**问题**: `useRemote ? remote : local` 作为 base 时，本地独有的 achievements/settings/dailyCheckIn 等会被远程覆盖。

**修复**: 采用深度合并 + 字段级取优策略，而非简单选择 remote/local 作为 base。

### 4.3 云数据库查询条件

**问题**: `uploadSave`/`downloadSave` 在 `where` 中显式指定 `_openid: openId`。若集合权限为 public，攻击者可伪造 openId 查询他人存档。

**修复**:
- 集合权限设为**仅创建者可读写**
- 查询条件只保留业务 `key`，移除 `_openid` 条件（服务端自动按创建者过滤）

### 4.4 验收标准

- [ ] 模拟场景：手机 A 解锁成就 X，手机 B 解锁成就 Y，云同步后两设备均可见 X+Y
- [ ] 云数据库权限截图确认：仅创建者可读写

## 5. 反作弊加固 🟡 P1

### 5.1 广告限制客户端化

**问题**: `adStats.rewardedToday` / `interstitialToday` 等全部存在本地 `localStorage`，用户可清空/修改存储绕过日上限和冷却。

**修复方案**:
- 客户端限制保留（防误触、防过度展示）
- 增加服务端辅助校验：每次广告观看后异步写入云数据库 `ad_records` 集合（至少作为对账依据）
- `AdManager` 增加 `BUILD.DISABLE_ALL_ADS` 总开关，审核期间一键切断所有广告调用

### 5.2 分享奖励可刷取

**问题**: `wx.shareAppMessage` 无真实完成回调，用户取消分享也会进入奖励分支。且 `lastShareDate` 基于本地时间，改系统日期可重置奖励。

**修复方案**:
- 接入 `withShareTicket: true`，分享后通过 `getShareInfo` 验证分享群/好友
- 奖励发放前增加云端分享次数校验（与云存档中的 `social.totalShares` 交叉验证）

### 5.3 验收标准

- [ ] 清空 localStorage 后广告日上限仍生效（依赖云端校验）
- [ ] 修改系统日期后分享奖励不重置

## 6. 性能优化 🟡 P1

### 6.1 StorageManager 频繁反序列化

**问题**: `GameManager.ts` 中 25 处直接调用 `StorageManager.load()`，几乎每个方法都重新反序列化完整存档对象。

**修复**: 引入内存缓存，减少反序列化开销。

```typescript
// GameManager.ts 增加局部缓存
private cachedSave: PlayerSaveData | null = null;

private getSave(): PlayerSaveData {
    if (!this.cachedSave) {
        this.cachedSave = StorageManager.load();
    }
    return this.cachedSave;
}

private invalidateSaveCache(): void {
    this.cachedSave = null;
}
```

**注意**: 缓存需在调用 `StorageManager.save()` 或外部修改存档后手动失效。

### 6.2 SDKManager setTimeout 泄漏

**问题**: `initialize()` 中的延迟任务（云初始化、广告预加载）使用不可取消的 `setTimeout`，场景切换或游戏退出时可能访问已释放对象。

**修复**:
```typescript
// SDKManager.ts
private static timeouts: number[] = [];

static initialize(): void {
    // ...
    this.timeouts.push(setTimeout(() => this.initCloudAndSync(), 1500));
    this.timeouts.push(setTimeout(() => AdManager.preloadAds(), 2500));
}

static cleanup(): void {
    this.timeouts.forEach(id => clearTimeout(id));
    this.timeouts = [];
}
```
