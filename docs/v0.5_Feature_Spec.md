> **文档版本**: v1.0
> **日期**: 2026-04-30
> **对应里程碑**: v0.5
> **目标**: 提升7日留存和DAU，让玩家每天都有回来的理由

## 0. 范围声明

### 0.1 In Scope（v0.5 必须实现）

- 每日签到正式版：动画、补签、连续加成
- 成就系统：10个成就配置、解锁动画、成就列表面板
- 主题商店：预览切换、解锁条件检查
- 每日挑战：随机关卡、步数排行榜

### 0.2 Out of Scope

- 主题颜色自定义（选色盘）— 后续版本
- 社交好友系统（除排行榜外）— 后续版本
- 赛季/活动系统 — 后续版本

### 0.3 广告 Mock 模式（v0.5 开发期特别说明）

**背景**：微信流量主需 UV ≥ 1000 才能开通，当前后台入口灰色，预计达到门槛前无法使用真实广告。

**方案**：`AdManager` 已内置 Mock 开关 `GAME_CONFIG.AD.MOCK_ENABLED`（默认 `true`）：
- **Mock 开启**：所有激励视频/插屏/Banner 调用均模拟成功，带 1.5s 延迟和 Toast 提示，统计正常写入存档
- **Mock 关闭**：走真实 `wx.createRewardedVideoAd`，需替换 `AD_UNIT_IDS` 为真实广告位 ID

**开发注意**：
- Mock 模式下 `totalAdsWatched` 仍会正常累加，VIP 主题解锁进度可直接测试
- 激励视频冷却时间、每日上限在 Mock 模式下**仍然生效**，防止开发期过度测试导致上限锁死
- 流量主开通后，只需两步切生产：
  1. 后台创建广告位，替换 `AD_UNIT_IDS` 中的占位符
  2. `GameConfig.ts` 中 `AD.MOCK_ENABLED = false`

### 1.4 补签功能

**触发条件**: 昨日未签到，且今日点击签到后弹窗不自动关闭

**规则**:
- 每次补签消耗：观看1次激励视频（`AdManager.showRewardedVideo('check_in_makeup')`）
- 补签后：`consecutiveDays` 连续（不中断），`checkInHistory` 对应位置标记为 true
- 补签上限：每周最多补签 2 次（防止刷连续天数）

**UI 文案**:
- 弹窗标题："补签昨日"
- 内容："观看视频即可补签昨天，保持连续签到记录不断！"
- 按钮："取消" / "看视频补签"

### 1.5 断签处理

**检测时机**: 每日首次启动，弹出签到弹窗前

**规则**:
- 如果 `lastCheckInDate` 不是昨天也不是今天 → 断签
- 断签后：`consecutiveDays` 重置为 0，`checkInHistory` 全部置 false
- UI 提示：弹窗顶部显示 "上次签到中断，从第1天重新开始"

## 4. 每日挑战 (DailyChallenge)

### 4.1 核心规则

- **关卡来源**: 从已生成的150关中每日随机抽取1关
- **固定每日**: 所有玩家当天挑战同一关（用日期做随机种子）
- **计分标准**: 完成步数越少排名越高
- **刷新时间**: 每日 0:00（北京时间）

**存档数据结构扩展**:
```typescript
export interface PlayerSaveData {
    // ... 原有字段 ...
    dailyChallenge: {
        lastChallengeDate: string;     // 上次挑战日期
        todayLevelId: number;          // 今日挑战关卡ID
        todayBestSteps: number;        // 今日最佳步数（0=未完成）
        consecutiveDays: number;       // 连续完成天数
        totalCompletions: number;      // 累计完成次数
    };
}
```

### 4.2 挑战关卡生成算法

```typescript
function getDailyChallengeLevelId(): number {
    const dateKey = WXAPI.getLocalDateKey(); // "2026-04-30"
    // 用日期字符串做简单哈希，确保同一天所有玩家得到相同结果
    let hash = 0;
    for (let i = 0; i < dateKey.length; i++) {
        hash = ((hash << 5) - hash) + dateKey.charCodeAt(i);
        hash |= 0;
    }
    // 映射到 1-150 关卡
    return (Math.abs(hash) % 150) + 1;
}
```

**倒计时计算**:
```typescript
function getTimeUntilMidnight(): string {
    const now = new Date();
    const tomorrow = new Date(now.getFullYear(), now.getMonth(), now.getDate() + 1);
    const diff = tomorrow.getTime() - now.getTime();
    const hours = Math.floor(diff / 3600000);
    const minutes = Math.floor((diff % 3600000) / 60000);
    const seconds = Math.floor((diff % 60000) / 1000);
    return `${String(hours).padStart(2, '0')}:${String(minutes).padStart(2, '0')}:${String(seconds).padStart(2, '0')}`;
}
```

### 4.4 挑战流程

```
玩家点击"开始挑战"
    │
    ▼
加载今日挑战关卡（与普通关卡相同的 gameplay 流程）
    │
    ▼
通关后弹出挑战结算
    - 显示本关步数
    - 如果打破个人最佳，显示"新纪录！"
    - 如果进入前100名，显示排名
    - 发放奖励（100金币 + 前100名额外钻石）
    │
    ▼
可选：分享成绩到朋友圈（炫耀）
```

**挑战结算与普通结算的区别**:
- 不显示"双倍奖励"按钮（避免与每日挑战奖励叠加）
- 显示"今日排名"（如果是前100名）
- 显示"我的历史最佳"对比
- 增加"再挑战一次"按钮（可无限重试刷步数）

### 4.5 排行榜数据

**数据存储**: 使用 `rank_data` 数据库集合（已在 CloudSaveManager 中预留）

**记录格式**:
```typescript
{
    _openid: string,       // 自动添加
    date: '2026-04-30',    // 挑战日期
    levelId: 87,
    steps: 12,             // 完成步数
    nickname: '玩家昵称',   // 从微信获取（如果有）
    updatedAt: Date.now(),
}
```

**查询前100名**:
```typescript
const db = wx.cloud.database();
const today = WXAPI.getLocalDateKey();
const res = await db.collection('rank_data')
    .where({ date: today })
    .orderBy('steps', 'asc')
    .limit(100)
    .get();
```

> **注意**：由于数据库权限"仅创建者可读写"的限制，客户端只能查询到自己的记录。要实现全服排行榜，需要：
> 1. 方案A：使用专用云函数查询排行榜（注意：V0.5.3 已恢复 `getOpenId` 身份云函数，但它只用于存档隔离，不用于全服排行榜）
> 2. 方案B：排行榜数据写入开放数据域（微信好友榜）
> 3. 方案C：排行榜降级为"个人最佳挑战记录"，不展示全服排名
>
> **建议采用方案C**：v0.5 先实现"个人最佳 + 今日步数"，全服排行榜留给 v0.6 或通过开放数据域实现。
