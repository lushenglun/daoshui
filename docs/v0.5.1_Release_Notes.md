## 1. 版本说明

v0.5.1 是 v0.5 功能开发完成后的**审核准备补丁**，不做任何玩法或运营功能新增，仅解决以下三类审核阻塞问题：

1. **微信主体认证未完成** → 无法发布
2. **Mock 广告入口暴露** → 审核员判定「功能虚假/体验不完整」
3. **GM 开发工具暴露** → 审核员判定「测试版/Demo」

## 2. 变更摘要

| 文件 | 变更 | 原因 |
|---|---|---|
| `assets/Scripts/Data/GameConfig.ts` | `VERSION: '0.5.1'`；新增 `BUILD` 配置块 | 集中管理发布/审核开关 |
| `assets/Scripts/Core/GameManager.ts` | GM 按钮、广告入口按 `BUILD` 开关条件渲染 | 审核合规 |

### 2.1 新增 BUILD 配置（GameConfig.ts）

```typescript
BUILD: {
    /** 是否为发布版本（true = 隐藏 GM 工具等开发功能） */
    IS_RELEASE: true,
    /** 审核期间是否隐藏所有广告入口（流量主未开通前建议开启） */
    HIDE_AD_ENTRIES_IN_REVIEW: true,
}
```

**说明**：
- `IS_RELEASE: true` → 设置面板中的 **GM 按钮** 不渲染
- `HIDE_AD_ENTRIES_IN_REVIEW: true` → 以下广告入口不渲染：
  - 主菜单「看视频领金币」按钮
  - 签到弹窗「看视频补签昨天」按钮
  - 游戏内金币不足选择面板中的「看视频」按钮（提示/撤销/双倍金币场景）

**流量主开通后恢复广告**：
```typescript
// GameConfig.ts
BUILD: {
    IS_RELEASE: true,
    HIDE_AD_ENTRIES_IN_REVIEW: false,  // ← 改这一行
}
```

## 3. 审核阻塞问题清单与解决方案

### 阻塞 1：微信主体认证未完成 ⭐ 最致命

**表现**：小程序后台提交审核时弹窗提示「当前账号未完成微信认证，将影响新版本发布」。

**后果**：
- 审核通过后也无法发布上线
- 影响主体信用分
- 必须完成认证才能走完全部流程

**解决**：
1. 登录[微信公众平台](https://mp.weixin.qq.com/)
2. 进入「设置 → 基本设置 → 微信认证」
3. 按主体类型提交资料：
   - 企业：营业执照 + 对公账户验证 + 300 元/年
   - 个体工商户：营业执照 + 经营者身份证 + 300 元/年
   - 个人：身份证 + 300 元/年
4. 等待审核（通常 1-3 个工作日）

---

### 阻塞 2：Mock 广告入口暴露

**表现**：游戏内存在「看视频得奖励」按钮，但点击后播放的是 1.5 秒模拟广告（Mock 模式），无真实广告内容。

**审核风险**：
- 审核员点击「看视频领金币」→ 看到假广告 → **以「功能虚假」驳回**
- 或判定为「功能不完整」，因为广告变现是核心玩法支撑但无法使用

**v0.5.1 解决方案**：
- `HIDE_AD_ENTRIES_IN_REVIEW: true` 时，所有广告入口按钮**不渲染**
- 保留非广告路径：金币购买提示/撤销、正常签到（无补签）、主题商店金币/钻石解锁

**流量主开通后恢复步骤**：
1. 微信后台「流量主」Tab 开通（需 UV ≥ 1000）
2. 创建广告位，获得真实 `adUnitId`
3. 替换 `AD_UNIT_IDS` 中的占位符
4. `GameConfig.ts` → `HIDE_AD_ENTRIES_IN_REVIEW: false`
5. `AD.MOCK_ENABLED: false`
6. 发版更新

---

### 阻塞 3：GM 开发工具暴露

**表现**：设置面板底部有「GM」按钮，可查看存档摘要、重置存档。

**审核风险**：
- 审核员看到 GM 面板 → **直接判定为「测试版/Demo」**
- 微信明确规则：「多次提交测试内容或 Demo，将受到相应处罚」

**v0.5.1 解决方案**：
- `IS_RELEASE: true` 时，设置面板中的 **GM 按钮不渲染**
- 开发者本地调试时，将 `IS_RELEASE: false` 即可恢复 GM 入口

## 4. 审核自检 Checklist

提交审核前，逐条确认：

### 4.1 基础合规

- [ ] **微信认证已完成**（后台无红色提示）
- [ ] **游戏类目审核已通过**（设置 → 基本设置 → 服务类目 → 游戏）
- [ ] **适龄提示（12+）已展示**：首次启动自动弹出，内容符合《微信小游戏适龄提示规范》
- [ ] **隐私协议入口已提供**：设置面板中有「隐私协议」按钮，可跳转（哪怕是占位页面）

### 4.2 功能完整性

- [ ] 第 1 关可正常进入并完成
- [ ] 选关页可正常浏览和跳转
- [ ] 暂停/设置/返回主菜单流程无报错
- [ ] 每日签到可正常领取奖励
- [ ] 主题商店可正常浏览和切换已解锁主题
- [ ] 成就列表可正常查看
- [ ] 每日挑战可正常进入并完成
- [ ] 分享功能可正常拉起（审核员不会真分享，但入口要可用）
- [ ] 排行榜可正常显示（编辑器/真机均可）

### 4.3 审核禁区

- [ ] **GM 按钮不可见**（设置面板底部没有 GM）
- [ ] **「看视频」按钮不可见**（主菜单、签到、游戏内均无）
- [ ] **无调试日志/红字报错**（开发者工具真机调试，控制台干净）
- [ ] **无「测试版」「Demo」「内测」等字样**
- [ ] **无未实现的空白按钮或占位文案**

### 4.4 包体验证

- [ ] 上传的是最新 V0.5 包（后台开发版本显示 V0.5）
- [ ] 真机体验版扫码测试通过（用预览二维码在真机上完整玩 3-5 关）
- [ ] 启动性能可接受（首屏 3 秒内出现）

## 6. 上线后切真实广告操作手册

### Step 1：拉 UV 开通流量主

- 游戏上线后，通过分享、自然传播、买量等方式积累独立访客（UV）
- 目标：UV ≥ 1000
- 达标后，后台「流量主」Tab 由灰变亮，点击开通

### Step 2：创建广告位

- 流量主后台 → 广告管理 → 新建广告位
- 创建三类广告位，记录 unitId：
  - 激励视频（建议创建 1 个通用位即可，多场景复用）
  - 插屏广告
  - Banner 广告

### Step 3：替换配置

修改 `assets/Scripts/Data/GameConfig.ts`：

```typescript
export const AD_UNIT_IDS = {
    rewardedVideo: 'adunit-你的真实ID',  // ← 替换
    interstitial: 'adunit-你的真实ID',   // ← 替换
    banner: 'adunit-你的真实ID',         // ← 替换
};

// BUILD 配置
BUILD: {
    IS_RELEASE: true,
    HIDE_AD_ENTRIES_IN_REVIEW: false,  // ← 改回 false，广告入口恢复显示
},

// AD 配置
AD: {
    MOCK_ENABLED: false,  // ← 改回 false，走真实广告
    // ... 其他配置不变
}
```

### Step 4：回归测试

- 开发者工具 → 真机调试
- 逐个验证广告点位：
  - 主菜单「看视频领金币」→ 拉起真实激励视频 → 看完获得金币
  - 游戏内提示/撤销/双倍金币 → 拉起真实激励视频
  - 签到补签 → 拉起真实激励视频
  - 关卡 6/9/12... → 自动触发插屏
  - 主菜单/选关/结算 → Banner 正常展示

### Step 5：发版更新

- Cocos Creator 构建 → 微信小游戏
- 微信开发者工具上传
- 后台提交版本更新审核
- 审核通过后发布
