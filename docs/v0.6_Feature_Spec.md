## 4. 好友助力（Friend Assist）

### 4.1 背景

微信小游戏的核心增长方式是社交裂变。当前分享仅能获得 50 金币（每日首次），缺乏「邀请新用户」的强激励。

### 4.2 助力机制

#### 助力场景 1：解锁限定主题「友谊之桥」

- **目标**: 邀请 3 位好友助力，解锁专属主题
- **流程**:
  1. 玩家 A 在主题商店点击「友谊之桥」→ 弹出「邀请 3 位好友助力解锁」
  2. 玩家 A 分享助力卡片到微信群/好友
  3. 好友 B 点击分享卡片进入游戏 → 自动记录为「已助力玩家 A」
  4. 玩家 A 累计 3 位不同好友助力后，主题自动解锁
- **防刷**: 同一好友只能助力 1 次；助力记录写入云数据库

#### 助力场景 2：每日求助（获得提示/撤销）

- **目标**: 卡关时向好友求助，获得免费提示或撤销次数
- **流程**:
  1. 玩家 A 在游戏中卡关 > 2 分钟 → 弹出「向好友求助？」
  2. 分享求助卡片
  3. 好友 B 点击进入 → 玩家 A 获得 1 次免费提示
  4. 每日限求助 3 次

## 5. 无限模式（Endless Mode）

### 5.1 背景

当前 150 关为固定关卡，核心玩家通关后缺乏长期内容。无限模式提供随机生成的可解关卡，延长游戏生命周期。

### 5.2 规则

- **入口**: 选关页增加「无限模式」标签页，或主菜单增加「无尽挑战」按钮
- **关卡生成**: 基于种子随机生成，每关参数：
  - 颜色种类: 4-6 种（随关卡数递增）
  - 瓶子数量: 6-10 个
  - 空瓶数量: 1-3 个
  - 容量: 固定 4 格
- **可解性保证**: 生成后立即用 BFS 验证可解，不可解则重新生成
- **难度递增**: 每 10 关增加 1 种颜色或减少 1 个空瓶
- **奖励**: 每通关 1 关奖励 30 金币（无星级，因为无 minSteps 基准）
- **排行榜**: 显示「无尽模式最高关卡数」好友排行

### 5.3 技术方案

```typescript
// LevelManager.ts 新增
static generateEndlessLevel(seed: number, depth: number): LevelConfig {
    // 基于 seed 确定性随机
    // depth: 已通关深度，控制难度
    const rng = new SeededRandom(seed + depth);
    const colorCount = Math.min(4 + Math.floor(depth / 10), 8);
    const bottleCount = Math.min(6 + Math.floor(depth / 20), 10);
    const emptyBottles = Math.max(3 - Math.floor(depth / 15), 1);
    
    // 生成初始状态
    let state = generateRandomState(rng, colorCount, bottleCount, emptyBottles);
    
    // 验证可解性（BFS，限制搜索深度 50 步）
    while (!isSolvable(state, 50)) {
        state = generateRandomState(rng, colorCount, bottleCount, emptyBottles);
    }
    
    return {
        levelId: 10000 + depth,  // 无限模式关卡 ID 从 10000 开始
        chapterId: 99,
        initialState: state,
        minSteps: 0,  // 无限模式无三星基准
        isEndless: true,
    };
}

// 存档扩展
interface PlayerSaveData {
    // ... 已有字段
    endless: {
        bestDepth: number;  // 历史最高通关深度
        currentDepth: number;  // 当前挑战深度
    };
}
```

## 6. 新手引导优化（Onboarding v2）

### 6.1 背景

当前前 10 关为教学关，但缺乏强制引导，部分新玩家不知道「双击自动倒水」「空瓶策略」等核心技巧，首日流失较高。

### 6.2 引导内容

| 关卡 | 引导时机 | 引导内容 | 交互方式 |
|---|---|---|---|
| 第 1 关 | 首次进入 | 「点击瓶子选中，再点击目标瓶倒水」 | 手指动画 + 高亮 |
| 第 1 关 | 倒水成功后 | 「把同色液体倒在一起就通关」 | Toast |
| 第 2 关 | 首次遇到空瓶 | 「空瓶可以临时存放任意颜色」 | 手指动画 + 高亮 |
| 第 3 关 | 首次可双击 | 「双击瓶子自动寻找目标」 | 脉冲提示 + 文字 |
| 第 5 关 | 首次使用提示 | 「金币不足？看视频获得提示」 | 弹窗 |
| 第 10 关 | 通关后 | 「解锁新主题！去商店看看吧」 | 红点提示 |

### 6.3 技术方案

}

## 10. 风险说明

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 音频团队交付延迟 | 中 | 先用免费音效库（如 freesound.org）做占位，真实资源到位后替换 |
| 流量主开通受阻（UV 不达标） | 高 | 先上非广告功能（活动、无限模式、音频），广告部分保持 Mock 待开通后切 |
| 端午节活动开发周期紧 | 中 | 若时间不够，降级为「仅限时主题 + 双倍金币」，去掉 3 关特殊关卡 |
| BFS 可解性验证性能差 | 低 | 限制搜索深度 50 步，超时即判定不可解重新生成；复杂状态生成概率低 |
| 好友助力云函数开发成本 | 中 | 复用 v0.5.3 的 `getOpenId` 云函数基础设施，最小化新增云函数 |
